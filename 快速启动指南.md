# BaseDocEditor 快速启动指南

## 🚀 快速开始

### 1. 启动开发服务器

```bash
# 确保已安装依赖
npm install

# 启动开发服务器
npm run dev
```

服务器将在 `http://localhost:5173` 启动。

### 2. 在飞书多维表格中测试

#### 步骤 1: 打开多维表格
- 在浏览器中打开你的飞书多维表格

#### 步骤 2: 添加扩展
1. 点击右上角的「扩展」按钮
2. 选择「添加扩展」
3. 选择「自定义扩展」或「开发者扩展」

#### 步骤 3: 配置开发地址
- 输入开发地址：`http://localhost:5173`
- 点击「确认」

#### 步骤 4: 开始使用
- 插件加载完成后即可使用！

## 📖 基本使用流程

### 1. 选择表格
- 在顶部下拉框中选择要编辑的表格
- 插件会自动加载该表格的所有记录和字段

### 2. 查看和编辑
- 每条记录以卡片形式展示
- 点击任意可编辑字段进入编辑模式
- 修改后点击「保存」或按 Enter 键

### 3. 追踪修改
- 修改的字段会以**黄色边框**高亮显示
- 右侧同步面板显示所有待同步的修改
- 可以查看修改前后的值对比

### 4. 同步到表格
1. 点击右侧面板的「同步到表格」按钮
2. 确认修改列表
3. 等待同步完成
4. 查看同步结果

## ⚙️ 支持的字段类型

| 字段类型 | 是否可编辑 | 说明 |
|---------|----------|------|
| ✅ 文本 | 可编辑 | 单行输入框 |
| ✅ 数字 | 可编辑 | 数字输入框 |
| ✅ 日期时间 | 可编辑 | 日期选择器 |
| ✅ 单选 | 可编辑 | 下拉选择 |
| ✅ 多选 | 可编辑 | 多选下拉 |
| ✅ 复选框 | 可编辑 | 复选框 |
| ❌ 公式 | 只读 | 计算字段 |
| ❌ 查找引用 | 只读 | 引用字段 |
| ❌ 自动编号 | 只读 | 系统字段 |

## ⌨️ 快捷键

- `Enter` - 保存当前编辑
- `Esc` - 取消编辑

## 💡 使用技巧

### 1. 批量编辑
- 可以一次性修改多个字段
- 所有修改会累积在同步面板中
- 点击一次「同步」即可批量提交

### 2. 撤销修改
- 在同步面板中，每个修改项右侧有删除按钮
- 点击可撤销单个字段的修改
- 或点击「清空」撤销所有修改

### 3. 草稿自动保存
- 所有修改会自动保存到浏览器本地
- 即使刷新页面，修改也不会丢失
- 草稿有效期为 7 天

## ⚠️ 注意事项

1. **权限要求**
   - 需要对多维表格有编辑权限
   - 只读字段无法编辑

2. **数据同步**
   - 同步前请仔细检查修改列表
   - 同步操作不可撤销
   - 建议先在测试表格中试用

3. **浏览器要求**
   - 推荐使用 Chrome、Edge 或 Safari
   - 需要启用 JavaScript

4. **性能建议**
   - 单次加载建议不超过 100 条记录
   - 大量数据可使用视图过滤

## 🐛 常见问题

### Q: 插件加载失败
**A:** 请检查：
- 开发服务器是否正常运行（`npm run dev`）
- 开发地址是否正确（`http://localhost:5173`）
- 浏览器控制台是否有错误信息

### Q: 修改没有同步
**A:** 请检查：
- 是否点击了「同步到表格」按钮
- 是否有足够的编辑权限
- 网络是否正常

### Q: 某些字段无法编辑
**A:** 以下字段类型不支持编辑：
- 公式字段
- 查找引用字段
- 自动编号字段
- 创建/修改时间/人字段

### Q: 草稿丢失了
**A:** 草稿保存在浏览器本地存储中，可能的原因：
- 清除了浏览器数据
- 超过了 7 天有效期
- 使用了隐私/无痕模式
