# BaseDocEditor 插件使用指南
## 目标读者
本指南面向已经在飞书工作区成功上线 BaseDocEditor 插件的业务人员、项目管理者与运营同学，帮助你在无需了解开发细节的情况下熟练完成文档批量生成、编辑与同步工作。
## 使用前Checklist
- 已在飞书多维表格后台完成插件安装并授予所需表权限
- 具备目标多维表格的编辑或更高权限
- 浏览器允许本地存储（用于草稿与追踪数据）
- 已准备好模板视图或过滤后的数据集以减少卡片数量
## 进入插件
1. 打开飞书多维表格并选择目标数据表
2. 点击右上角`扩展`→`添加扩展`→选择已发布的 BaseDocEditor
3. 首次进入需授权，点击允许即可
4. 插件加载完成后会显示“表格选择器+文档区+同步面板”的三栏布局
## 任务全览
1. 选择表格与视图，确认数据范围
2. 浏览文档卡片并进行字段编辑
3. 在同步面板核对修改记录
4. 提交同步，等待结果提示
5. 视需要刷新或切换下一批数据
## 表格与数据选择
- 顶部下拉框展示你拥有权限的表；选择后会自动加载字段与记录
- 可通过飞书原生视图筛选数据以减少一次性加载量，建议不超过100条
- 若表结构更新，使用右上角`刷新数据`重新拉取字段描述
## 文档渲染区
- 每条记录会渲染成独立卡片，字段按网格布局排列
- 只读字段（公式、查找引用、系统字段等）以灰色标注并不可编辑
- 拖动页面右侧滚动条以在多卡片间快速定位
## 字段编辑
1. 点击任意可编辑字段进入编辑态
2. 根据字段类型出现对应输入控件（文本框、数字框、日期选择、单/多选等）
3. `Enter`保存当前字段，`Esc`撤销本次修改
4. 保存后字段会以黄色描边显示，提示已产生待同步修改
5. 未提交前所有修改被写入浏览器本地草稿（默认保留7天）
## 修改追踪面板
- 面板顶部展示待同步数量徽章与批量操作按钮
- 列表项包含字段名、原值、现值与记录定位按钮
- 点击单条记录的`撤销`可删除该修改并恢复原值
- `清空全部`用于放弃本轮所有改动
## 同步到多维表格
1. 点击`同步到表格`按钮进入确认弹窗
2. 再次核对修改列表，确认无误后提交
3. 同步过程中保持页面停留，状态指示会展示正在写入的记录
4. 成功后所有字段恢复正常边框并自动刷新最新数据；失败项会保留在面板中供重试
## 安全与权限提示
- 仅在具有编辑权限的表格内同步，若权限不足会直接阻止操作
- 同步为批量写入动作，提交后无法整体撤销，必要时请先在测试表验证
- 草稿存储依赖浏览器，清除缓存或更换设备会导致未同步修改丢失
## 性能最佳实践
- 大批量记录请拆分视图分批处理，避免一次加载过多卡片造成卡顿
- 复杂模板可优先在`TemplatePage`中预设版式，减少在线编辑时间
- 同步失败大多与网络或字段校验有关，优先检查必填字段与数据类型
## 常见问题
|问题|解决方案|
|---|---|
|无法进入插件|确认所在多维表格已安装扩展并刷新页面|
|字段不可编辑|检查字段类型是否受支持或当前账号是否仅有只读权限|
|同步失败|查看面板中的错误提示，常见原因包括网络中断、必填校验失败或字段被他人删除|
|草稿丢失|确认未清理浏览器缓存，或在同一设备与浏览器内继续操作|
## 建议的日常流程
1. 开始前刷新数据，确保字段结构最新
2. 先使用视图过滤出当日需处理的记录
3. 按卡片逐条审核与编辑，实时留意黄色高亮
4. 编辑结束后集中查看同步面板做最终确认
5. 同步完成记录本次操作时间，便于后续追溯
